# DevOps Technical Challenge

## Overview
This repository contains a **broken** Java application with issues in its **Maven build**, **Docker setup**, and **CI/CD pipeline**.  
Your task is to **analyze, debug, and fix** the problems.

---

## Tasks

1. **Fix the Build 🛠️**  
   - The application does not build correctly. Identify and fix the issue in `pom.xml`.
   - 🏷️ **Create a branch**: `fix-maven-build`

2. **Fix the Dockerfile 🐳**  
   - The `Dockerfile` has misconfigurations. Adjust it to correctly run the application.
   - 🏷️ **Create a branch**: `fix-dockerfile`

3. **Fix the CI/CD Pipeline 🚀**  
   - The GitHub Actions workflow contains an issue. Find and resolve it.
   - 🏷️ **Create a branch**: `fix-ci-pipeline`

4. **Run and Verify 🔥**  
   - Ensure the application builds, runs, and responds on `http://localhost:8080/api/hello`.

---

### Bonus (Optional)
- Improve the **Dockerfile**.
- Enhance the **CI/CD pipeline**.

### Criteria for Submission 📌
- Each fix must be committed in a **separate branch** as specified above.
- After fixing, **merge each branch into `main`** via a pull request.
- Keep commit messages **clear and descriptive**.

Good luck! 🚀
